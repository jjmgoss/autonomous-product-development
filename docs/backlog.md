# Backlog

Track remaining work in a structured way.

## Suggested sections

- active now
- next up
- blocked by missing evidence
- deferred
- rejected
- technical debt
- future experiments

## Backlog rules

- Keep the backlog honest.
- If something is not being worked, do not pretend it is active.
- Research, validation, and offer-test tasks belong here when they are the critical path.
- Rejected items should keep a short reason so the same weak idea is not revived accidentally.
