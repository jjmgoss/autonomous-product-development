# Roadmap

Break the work into milestones and execution phases.

## Suggested sections

- milestone overview
- first vertical slice
- milestone goals and acceptance checks
- dependencies and unresolved risks
- cut-line options
- later improvements only if still credible

## Planning rules

- The first milestone should deliver something runnable or meaningfully testable, not just scaffolding.
- Each milestone should advance one real user-visible capability.
- If a milestone mainly exists to prepare for future milestones, it is probably too abstract.
