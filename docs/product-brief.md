# Product Brief

Use this file once discovery has selected one validated opportunity and the loop moves into build-forward mode.

## Purpose

Describe the chosen product clearly enough that a reviewer can understand the buyer, workflow, wedge, prototype goal, and scope boundary before code is written.

## Include

- product name or working title
- prototype class: UI shell, working demo, or real local prototype
- first buyer or user
- painful workflow
- job to be done
- first wedge
- prototype success event
- explicit MVP scope
- explicit non-goals
- credible first monetization path
- why this is not a broad platform fantasy
- why agent-led build and operation are plausible
- top risks and adoption barriers
- what is likely to stay stubbed in the first prototype

## Rule

The product brief should make it obvious what the first thin product slice is, what success means, and what should not be built yet.
