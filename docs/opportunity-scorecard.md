# Opportunity Scorecard

Score candidate ideas before choosing one.

## Suggested dimensions

Use a 1–5 scale.

- pain intensity
- frequency of problem
- evidence quality
- prototype feasibility
- differentiation potential
- monetization plausibility
- technical risk
- distribution difficulty

## Output format

For each opportunity, record:

- short name
- target user
- one-sentence problem statement
- scores by dimension
- short explanation
- final recommendation

## Rule

Do not pick the flashiest idea by default.
Pick the one with the strongest combination of pain, evidence, and tractable scope.
