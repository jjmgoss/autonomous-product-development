# Launch Checklist

Use this checklist if the prototype is far enough along to be exposed to users.

## Product readiness

- [ ] core workflow works end to end
- [ ] scope is narrow and understandable
- [ ] release notes are current

## Quality readiness

- [ ] key flows have been exercised
- [ ] obvious failures are documented
- [ ] known gaps are listed

## Operational readiness

- [ ] basic observability exists or the lack is documented
- [ ] deployment steps are written down
- [ ] rollback or rollback-like recovery is considered

## Business and support readiness

- [ ] the target user and wedge are still clear
- [ ] the monetization or offer angle is documented
- [ ] the likely support burden is realistic for a solo operator
- [ ] cost or service dependencies are understood well enough for this level of exposure

## Human readiness

- [ ] someone can explain what this product is for
- [ ] someone can explain why this version is worth showing
