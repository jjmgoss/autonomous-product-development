# Release Notes and Verification

Use this file before claiming the MVP is ready.

## Include

- what the release includes
- whether the result is a UI shell, working demo, or real local prototype
- what was verified
- what tests or smoke checks were run
- what health or sanity check exists
- what is stubbed or fake
- known issues
- gaps versus requirements
- deployment status
- next milestone recommendation

## Rule

This file must be honest enough that a human reviewer can tell whether the product is truly usable.
