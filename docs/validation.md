# Validation

Use this file to determine whether the chosen idea deserves to be built.

## Include

- strongest evidence supporting the idea
- strongest reasons the idea may be weak
- competitor/substitute analysis
- what users would need to switch or care
- the smallest prototype that could falsify the thesis
- explicit go/no-go recommendation

## Rule

A go decision should be earned, not assumed.
If the idea is weak, say so.
