# Debugging Skill

## Goal
Efficiently identify and resolve defects.

## When to Use
When encountering unexpected behavior or test failures.

## Procedure
1. Reproduce the issue.
2. Isolate the root cause.
3. Apply the smallest fix.
4. Add or update tests as needed.
5. Document the fix in `docs/work-log.md`.

## Expected Outputs
- Bug fix
- Test evidence
- Work log entry

## Common Failure Modes
- Guessing without evidence
- Fixing symptoms, not causes
