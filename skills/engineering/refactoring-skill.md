# Refactoring Skill

## Goal
Improve code structure without changing behavior.

## When to Use
When code is hard to understand, maintain, or extend.

## Procedure
1. Identify code smells or complexity.
2. Plan small, safe refactors.
3. Run tests before and after changes.
4. Update docs if needed.

## Expected Outputs
- Cleaner code
- Passing tests
- Updated docs

## Common Failure Modes
- Refactoring without tests
- Large, risky changes
