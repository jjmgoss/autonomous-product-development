# Implementation Loop Skill

## Goal
Disciplined, incremental implementation of the MVP.

## When to Use
During all coding phases.

## Procedure
1. Restate the issue and acceptance criteria.
2. Inspect the codebase before changing it.
3. Identify the smallest viable change.
4. Implement narrowly.
5. Run targeted checks first, then broaden testing.
6. Update docs if behavior or scope changed.

## Expected Outputs
- Code changes
- `docs/work-log.md`
- `docs/decision-log.md`

## Common Failure Modes
- Overbuilding
- Skipping verification
