# Testing Skill

## Goal
Verify that code meets requirements and is robust.

## When to Use
After each implementation step and before release.

## Procedure
1. Write targeted tests for new code.
2. Run all relevant tests.
3. Document test results in `docs/release.md`.

## Expected Outputs
- Test results
- Release evidence

## Common Failure Modes
- Incomplete test coverage
- Not documenting test results
