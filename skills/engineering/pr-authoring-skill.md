# PR Authoring Skill

## Goal
Create clear, actionable pull requests for review.

## When to Use
When submitting code for review.

## Procedure
1. Summarize the change and rationale.
2. Link to relevant docs and issues.
3. List acceptance criteria and test evidence.
4. Highlight risks or open questions.

## Expected Outputs
- Well-documented PR
- Linked issues/docs

## Common Failure Modes
- Vague PR descriptions
- Missing test or doc links
