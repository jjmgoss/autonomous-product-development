# Code Review Skill

## Goal
Ensure code quality, maintainability, and alignment with requirements.

## When to Use
Before merging any code changes.

## Procedure
1. Review code for correctness, clarity, and adherence to requirements.
2. Check for updated docs and tests.
3. Provide actionable feedback.

## Expected Outputs
- Reviewed code
- Feedback or approval

## Common Failure Modes
- Superficial review
- Missing doc/test updates
