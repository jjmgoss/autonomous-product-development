# Roadmap Skill

## Goal
Break work into milestones and issue-sized slices.

## When to Use
After design, before implementation.

## Procedure
1. Define milestones for the MVP.
2. Decompose work into issues and backlog items.
3. Ensure first milestone is a runnable/testable vertical slice.

## Expected Outputs
- `docs/roadmap.md`
- `docs/backlog.md`

## Common Failure Modes
- Planning only scaffolding, not vertical slices
- Overloading the first milestone
