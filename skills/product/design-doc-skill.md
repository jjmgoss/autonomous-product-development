# Design Doc Skill

## Goal
Create a simple, credible technical design for the MVP.

## When to Use
After requirements are defined.

## Procedure
1. Outline architecture overview and key flows.
2. Define data model and interface expectations.
3. Note deployment, security, and test strategy.

## Expected Outputs
- `docs/design.md`

## Common Failure Modes
- Overcomplicating the design
- Missing test or security considerations
