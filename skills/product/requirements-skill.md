# Requirements Skill

## Goal
Translate the validated idea into explicit, testable requirements.

## When to Use
After validation, before design.

## Procedure
1. Define target user and problem statement.
2. Write user stories, functional and non-functional requirements.
3. List acceptance criteria and non-goals.

## Expected Outputs
- `docs/requirements.md`

## Common Failure Modes
- Vague requirements
- Missing acceptance criteria
